# Useful Hour Scorecard

Use this to judge whether a heartbeat run was valuable.

## Score Each 0-2

- Artifact: Did the run leave a durable file, script, commit, eval, or source-backed note?
- Verification: Did it run a concrete check or record why none applied?
- Learning: Did it add new evidence or clarify an assumption?
- Steering: Did it choose a narrow next step and avoid repeating stale work?
- Handoff: Can the next run resume in under five minutes?
- Restraint: Did it avoid scope creep, unsafe actions, and prompt bloat?

## Interpretation

- 10-12: excellent hour
- 7-9: useful hour
- 4-6: salvageable but needs a sharper next action
- 0-3: busywork; improve the automation prompt before continuing

## Minimum Bar

Every run should score at least 7 unless it was blocked by a real external dependency.
